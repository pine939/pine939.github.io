# TDD 리뷰 

TDD는 테스트를 많이 작성하는 의미가 아니라 Test Case를 개발 전에 만들고, 이 Test Case가 통과 되도록 기능을 개발하는 방법론을 말한다. 테스트를 작성하지 않은 코드는 구현하지 않는 것이 원칙이다. 

1. Describe it. (RDD : Readme Driven Development)
2. Make it fail.
3. Make it green.
4. Refactoring.
